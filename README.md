# BPMN Editor

A modern, versatile BPMN 2.0 diagram editor that runs everywhere.
Built with **Vite**, **bpmn-js**, and **Electron**, it works as a **Progressive Web App (PWA)** and a **Native Desktop Application**.

## üöÄ Features

*   **BPMN 2.0 Support:** Full support for viewing and editing BPMN 2.0 diagrams.
*   **Cross-Platform:**
    *   **Web / PWA:** Installable directly from the browser (Works offline!).
    *   **Desktop:** Native Installer for Windows (MSI/EXE) via Electron.
*   **Local File System:** Optimized for local file editing with "Save" and "Save As" support.
*   **Drag & Drop:** Drag `.bpmn` or `.xml` files directly onto the canvas to open them.
*   **Export:** Export diagrams as `.bpmn` (XML), `.svg`, or **High-Quality PNG** images.
*   **Minimap:** Built-in minimap for navigating large diagrams.
*   **Native Color Support:** Style elements with custom colors.

### Desktop App (Windows)
To create the installer (MSI/EXE):
```bash
npm run dist
```
*The installer files will be generated in the `dist` or `release` folder.*

### Web App (PWA) on GitHub Pages
To deploy the live version to `https://bpmn.veilands.com`:
```bash
npm run deploy
```
*   **Note:** This command automatically handles building the `dist` folder, adding the `.nojekyll` file (for GitHub compatibility), and pushing to the `gh-pages` branch.
*   **Important:** Remember to push your source code changes to `main` separately (`git push origin main`).

## ‚ùì FAQ

**Why isn't the "Export PNG" button working?**
Ensure you have a valid diagram loaded. The button is disabled if the canvas is empty.

**Why does the PWA Install prompt look for `manifest.webmanifest`?**
This file is auto-generated by Vite during the build process based on `vite.config.js`. You do not need to edit it manually.

**How do I change the App Icon?**
Replace `public/icon.png` (and update references in `vite.config.js` if the name changes).
